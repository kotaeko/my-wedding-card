## 1단계: content.js 나머지 채우기

### 계좌번호 입력

`content.js`의 `accounts` 항목을 채우세요:

```js
accounts: {
  groom: [
    { role: "신랑", bank: "카카오뱅크", number: "3333-00-0000000", holder: "고태혁" },
    { role: "아버지", bank: "은행명", number: "계좌번호", holder: "고명덕" },
    { role: "어머니", bank: "은행명", number: "계좌번호", holder: "정효임" },
  ],
  bride: [
    { role: "신부", bank: "은행명", number: "계좌번호", holder: "김서아" },
    ...
  ],
},
```

### 교통 정보 (버스 / 자가용)

`transportInfo` 항목에서 빈 `detail` 을 채우세요:

```js
{ icon: "🚌", label: "버스", detail: "버스 노선 및 정류장 정보" },
{ icon: "🚗", label: "자가용", detail: "주차 안내 또는 내비 검색어" },
```

> 정보가 없으면 해당 줄 통째로 지워도 됩니다.

## 2단계: Firebase 설정 (하객 사진 업로드 + RSVP용)

1. https://console.firebase.google.com 접속 → 구글 계정으로 로그인
2. **프로젝트 만들기** → 이름 입력 (예: `my-wedding-2026`)
3. 왼쪽 메뉴 **빌드 > Storage** → 시작하기 → 테스트 모드로 시작
4. 왼쪽 메뉴 **빌드 > Firestore Database** → 데이터베이스 만들기 → 테스트 모드
5. 왼쪽 아이콘(톱니바퀴) **프로젝트 설정** → 아래로 스크롤 → **내 앱** → `</>` (웹) 클릭
6. 앱 닉네임 입력 후 **앱 등록** → 아래 코드에서 값들 복사

`firebase-config.js` 파일을 열어 빈칸에 붙여넣기:

```js
const FIREBASE_CONFIG = {
  apiKey: "여기에 붙여넣기",
  authDomain: "여기에 붙여넣기",
  projectId: "여기에 붙여넣기",
  storageBucket: "여기에 붙여넣기",
  messagingSenderId: "여기에 붙여넣기",
  appId: "여기에 붙여넣기",
};
```

## 3단계: OG 이미지 만들기 (카카오톡 공유 미리보기)

링크를 카톡으로 보낼 때 뜨는 미리보기 이미지예요.

1. 캔바(canva.com)에서 **1200 × 630** 사이즈로 이미지 만들기
   - 커플 사진 + 날짜 + 이름 넣으면 좋아요
2. https://cloudinary.com 무료 가입
3. **Media Library** → 이미지 업로드
4. 업로드된 이미지 클릭 → **Copy URL** 복사
5. `content.js`에서:

```js
ogImageUrl: "https://res.cloudinary.com/여기에붙여넣기",
```

## 4단계: Netlify 배포

1. https://netlify.com 접속 → 무료 가입
2. **Sites** 탭 → 아래 영역에 `모바일청첩장` 폴더 통째로 드래그
3. 몇 초 후 주소 생성 (예: `random-name-123.netlify.app`)
4. **도메인 변경** (선택): Site settings → Domain management에서 이름 변경 가능

### 내용 수정 후 다시 배포하는 법
`content.js` 수정 → 저장 → Netlify 사이트 열기 → **Deploys** 탭 → 폴더 다시 드래그

---

## 하객 사진 / RSVP 확인하는 법

배포 후 하객들이 사용하면 Firebase에서 확인 가능해요.

1. https://console.firebase.google.com 접속
2. 프로젝트 선택
3. **Storage** : 하객이 올린 사진 파일 확인 및 다운로드
4. **Firestore** 클릭
   - `guestPhotos` 컬렉션: 하객 사진 목록 (이름, 업로드 시간)
   - `rsvp` 컬렉션: 참석 의사 목록 (이름, 참석 여부, 인원, 식사 여부)

## 자주 있는 문제

| 증상 | 원인 | 해결 |
|---|---|---|
| 사진이 안 보임 | 파일 ID 오타 또는 공유 설정 안 됨 | 드라이브 공유 설정 확인. URL은 반드시 `lh3.googleusercontent.com/d/파일ID` 형식 사용 |
| 사진이 처음엔 보이다 사라짐 | 구글 트래픽 한도 초과 | 잠시 후 새로고침 |
| 하객 업로드 안 됨 | firebase-config.js 미설정 | 2단계 다시 확인 |
| 지도 버튼이 안 열림 | 지도 URL이 비어있음 | content.js의 venueMapUrl / naverMapUrl 확인 |
| 이름이 "신랑 이름"으로 표시 | content.js 미수정 | 1단계 진행 |
